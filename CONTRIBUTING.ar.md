## 🚀 كيفية المساهمة

### الخطوة ١: Fork المستودع

```bash
# من GitHub
1. اذهب إلى github.com/arabic-skills-library/arabic-skills-library
2. اضغط على Fork
3. استنسخ النسخة المحلية:
git clone https://github.com/YOUR-USERNAME/arabic-skills-library.git
```

### الخطوة ٢: اختر مهارة أو أنشئ جديدة

```bash
# تصفح المهارات الموجودة
ls skills/

# أو اختر فئة جديدة
cd skills/[category]/
```

### الخطوة ٣: اتبع القالب

استخدم القالب الموحد في `SKILL_TEMPLATE.md`:

```markdown
---
name: skill-name
name_ar: اسم المهارة بالعربية
description: وصف موجز
category: [business|government|education|developer|healthcare|media|security|linguistics]
language: ar
version: 1.0.0
author: github-username
---

## الهدف (Purpose)
...
```

### الخطوة ٤: اختبر المهارة

```bash
# تأكد من صحة التنسيق
# اختبر المهارة مع LLM المفضل لديك
```

### الخطوة ٥: قدم Pull Request

```bash
git add skills/[category]/[skill-name]/SKILL.md
git commit -m "feat: إضافة مهارة [اسم المهارة]"
git push origin main
# أنشئ Pull Request على GitHub
```

## 📝 أنواع المساهمات

### ١. إنشاء مهارات جديدة ⭐⭐⭐

**المستوى**: متقدم  
**النقاط**: ١٠ نقاط

**المتطلبات**:
- بحث شامل عن الموضوع
- أمثلة عملية واقعية
- كلمات مفتاحية شاملة
- مراجع موثوقة

### ٢. تحسين مهارات موجودة ⭐⭐

**المستوى**: متوسط  
**النقاط**: ٥ نقاط

**الأمثلة**:
- إضافة أمثلة إضافية
- تحسين الوضوح
- تحديث المراجع
- إصلاح أخطاء إملائية

### ٣. مراجعة المهارات ⭐⭐

**المستوى**: متوسط  
**النقاط**: ٣ نقاط لكل مراجعة

**المتطلبات**:
- التحقق من الدقة التقنية
- مراجعة اللغة العربية
- اختبار المهارة عملياً

### ٤. ترجمة ⭐

**المستوى**: مبتدئ  
**النقاط**: ٢ نقطة

**الأمثلة**:
- ترجمة الأقسام الإنجليزية
- تعريب المصطلحات التقنية

### ٥. تقارير الأخطاء ⭐

**المستوى**: مبتدئ  
**النقاط**: ١ نقطة

**من خلال Issues على GitHub**

☐ اختبرت المهارة بنجاح
☐ راجعت الإملاء والنحو العربي

## 🔍 عملية المراجعة

### المراحل

```
١. الإرسال
   │
   ▼
٢. فحص تلقائي (Format Check)
   │
   ▼
٣. مراجعة تقنية (Technical Review)
   │
   ▼
٤. مراجعة لغوية (Language Review)
   │
   ▼
٥. الموافقة النهائية (Final Approval)
   │
   ▼
٦. النشر (Publication)
```

### وقت المراجعة المتوقع

| المرحلة | الوقت المتوقع |
|---------|--------------|
| الفحص التلقائي | فوري |
| المراجعة التقنية | ٢-٣ أيام |
| المراجعة اللغوية | ١-٢ يوم |
| الموافقة النهائية | ١ يوم |
| **المجموع** | **٥-٧ أيام** |

### حالات المراجعة

#### ✅ مقبول
- جميع المعايير محققة
- سيندمج مباشرة

#### ⚠️ مقبول بشروط
- يحتاج تعديلات طفيفة
- سيعاد المراجعة بعد التعديل

#### ❌ مرفوض
- مشاكل جوهرية
- يحتاج إعادة كتابة
- سيتم توضيح الأسباب

## ❓ أسئلة شائعة

### س: هل يجب أن تكون خبيراً في الموضوع؟
**ج**: يفضل، لكن يمكنك البحث والاستشهاد بمصادر موثوقة.

### س: كم يجب أن يكون طول المهارة؟
**ج**: لا يوجد حد، لكن الأفضل ١٠٠٠-٣٠٠٠ كلمة لتغطية شاملة.

### س: هل يمكنني المساهمة بمهارات بالعامية؟
**ج**: نعم! لدينا قسم خاص للهجات في `dialects/`.

### س: كيف أختار موضوعاً؟
**ج**: تصفح `docs/skill-wishlist.md` للمواضيع المطلوبة.

### س: هل يمكنني تحديث مهاراتي بعد النشر؟
**ج**: نعم! نشجع على التحسين المستمر.

## 🎯 البدء السريع

```bash
# ١. Fork
# ٢. Clone
git clone https://github.com/YOUR-USERNAME/arabic-skills-library.git

# ٣. اختر موضوعاً
cd skills/developer/

# ٤. أنشئ مجلد جديد
mkdir arabic-new-skill
cd arabic-new-skill

# ٥. انسخ القالب
cp ../../SKILL_TEMPLATE.md SKILL.md

# ٦. اكتب المهارة
# افتح SKILL.md وعدّله

# ٧. أرسل
git add .
git commit -m "feat: إضافة مهارة [الاسم]"
git push
# أنشئ Pull Request
```

## 📜 المدونة السلوكية

### قيمنا

1. **الاحترام**: تعامل مهذب مع جميع المساهمين
2. **الجودة**: لا تقبل إلا الأفضل
3. **التعاون**: ساعد الآخرين على النجاح
4. **الشفافية**: كن واضحاً في المراجعات
5. **التنوع**: رحّب بجميع الخلفيات والآراء

### غير مقبول

- التعليقات غير المهذبة
- الانتحال العلمي
- المحتوى المنحاز أو المسيء
- الترويج التجاري غير المصرح به
